# Nova Mutation-Guided Testing

> Meta의 ACH(Automated Compliance Hardening)에서 영감. LLM으로 뮤턴트를 생성하고,
> 살아남은 뮤턴트를 기반으로 테스트를 보강한다.

## 프로세스

### Step 1: 뮤턴트 생성
대상 코드에서 의미 있는 뮤테이션을 생성한다:
- **조건 반전**: `if (x > 0)` → `if (x <= 0)`
- **경계값 변경**: `x >= 10` → `x >= 11`
- **반환값 변경**: `return true` → `return false`
- **null 삽입**: 정상 반환 → `null` 반환
- **예외 제거**: try-catch 블록의 catch 제거

### Step 2: 기존 테스트로 뮤턴트 사냥
- 각 뮤턴트에 대해 기존 테스트를 실행
- **죽은 뮤턴트**: 테스트가 실패 → 기존 테스트가 잘 잡고 있음 ✅
- **살아남은 뮤턴트**: 테스트가 여전히 통과 → 테스트 커버리지 갭 ⚠️

### Step 3: 살아남은 뮤턴트 기반 테스트 보강
- 살아남은 뮤턴트가 발견한 커버리지 갭에 대한 테스트를 작성
- 각 새 테스트는 해당 뮤턴트를 죽이는지 확인

### Step 4: 리포트
```
━━━ Mutation Test Report ━━━━━━━━━━━━━━━━━━
  대상: {파일/디렉토리}
  총 뮤턴트: {N}개
  죽은 뮤턴트: {N}개 ({N}%)
  살아남은 뮤턴트: {N}개 ({N}%)

  살아남은 뮤턴트 목록:
  1. {파일:라인} — {뮤테이션 설명}
  2. {파일:라인} — {뮤테이션 설명}

  보강된 테스트: {N}개
  최종 뮤턴트 사망률: {N}%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 적용 시점
- `/auto` Phase 5 Evaluator에서 선택적으로 실행
- `/review` 코드 리뷰 시 테스트 커버리지 심화 분석
- 직접 호출: 스킬로 invoke
